## TanStack Query Keys & Hooks (`lib/queries/`)

```typescript

export const queryKeys = {
  auth: {
    user: ['auth', 'user'] as const,
  },
  movies: {
    search: (query: string, page: number) => ['movies', 'search', { query, page }] as const,
    detail: (imdbID: string) => ['movies', 'detail', imdbID] as const,

    detail: (id: string, page: number, limit: number) => ['watchlists', 'detail', id, { page, limit }] as const,
    search: (query: string) => ['watchlists', 'search', query] as const,
  },
};
```

## Auth Flow Implementation

1. **Login Page** → `GoogleLoginButton` → `credentialResponse.credential` (ID token)
2. **POST /auth/google/login** → Backend sets httpOnly cookies
3. **Redirect to `/dashboard`**
4. **AuthProvider** on mount: `queryClient.invalidateQueries({ queryKey: ['auth', 'user'] })`
5. **useUser()** hook → `GET /auth/me` (decode JWT client-side from cookie) OR add lightweight `/auth/me` endpoint
6. **All requests** → cookies sent automatically
7. **401** → Interceptor calls `POST /auth/refresh` → retry once → if fails, logout + redirect to `/login`
8. **Logout** → `POST /auth/logout` → clear context → redirect `/login`

> **Note**: Backend has no `/auth/me`. Option A: Decode JWT client-side (access token in cookie not accessible via JS). Option B: Add `GET /auth/me` to backend (recommended - 5 min work). I'll assume Option B.

## Anti-Patterns to Avoid

| Don't | Do |
|-------|----|
| Use `useEffect` for data fetching | Use TanStack Query |
| Store tokens in localStorage | Use httpOnly cookies (backend handles) |
| Direct axios calls in components | Use query hooks |
| Mutate cache manually without rollback | Use `onMutate`/`onError`/`onSettled` |
| Hardcode API URLs | Use `NEXT_PUBLIC_API_URL` |
| Skip loading/error states | Every query has skeleton + error toast |
| Forget `withCredentials: true` | Set once in axios instance |
